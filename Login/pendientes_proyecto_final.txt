PENDIENTES DEL PROYECTO FINAL - 4eng-Login
Fecha: 2026-05-11
Estado: En progreso

Objetivo de este documento
- Dejar claro que falta para llegar a una version final segura y lista para produccion.
- Ordenar tareas por prioridad para que el equipo avance sin bloquearse.

==================================================
1) PENDIENTES CRITICOS (HACER PRIMERO)
==================================================

1.1 Seguridad de contrasenas (OBLIGATORIO)
Estado actual:
- En la tabla users, las contrasenas estan en texto plano (campo password).

Riesgo:
- Si se filtra la base, se exponen contrasenas reales.

Que falta hacer:
- Implementar hash de contrasena en backend (Argon2id recomendado, bcrypt como alternativa).
- Guardar solo hash en password_hash.
- Dejar de usar password para login.
- Migrar cuentas de prueba a hash.
- Luego eliminar o inutilizar columna password.

Resultado esperado:
- En la base no se ve ninguna contrasena legible.

--------------------------------------------------
1.2 Cerrar exposicion de phpMyAdmin por defecto
Estado actual:
- phpMyAdmin fue accesible desde internet y despues se deshabilito (404), lo cual es correcto para reposo.

Que falta hacer:
- Definir politica oficial:
  - Por defecto: desactivado.
  - Solo mantenimiento: activarlo temporalmente.
- Cuando este activo: proteger con HTTPS + autenticacion extra (basic auth) y/o restriccion por IP.

Resultado esperado:
- Exposicion minima de superficie de ataque.

--------------------------------------------------
1.3 Login/registro no deben depender de conexion directa Qt -> MySQL
Estado actual:
- Qt consulta DB directa para login y registro.

Riesgo:
- Credenciales de DB en cliente.
- Logica de seguridad distribuida y dificil de controlar.

Que falta hacer:
- Mover login y registro a FastAPI.
- Qt debe llamar endpoints HTTP del backend.
- Backend valida, hashea, audita y responde token/sesion.

Resultado esperado:
- Arquitectura cliente-servidor segura y mantenible.

==================================================
2) PENDIENTES ALTOS (SIGUIENTE BLOQUE)
==================================================

2.1 Configurar FastAPI base de autenticacion
Que falta hacer:
- Endpoints minimos:
  - POST /auth/register-user
  - POST /auth/register-company
  - POST /auth/login
- Validaciones:
  - username y email unicos
  - role permitido (Usuario, Empresa, Admin)
  - reglas de datos requeridos por tipo
- Integracion con tabla users + perfiles.

2.2 Configurar Nginx delante de FastAPI
Que falta hacer:
- Reverse proxy a FastAPI.
- TLS (Let's Encrypt) en dominio.
- Redireccion HTTP -> HTTPS.
- Reglas basicas de hardening.

2.3 Verificar conexion real Qt con backend
Que falta hacer:
- Reemplazar consultas SQL en Qt por llamadas a API.
- Manejo de errores de red, timeout y reintentos.
- Mensajes de estado claros al usuario.
