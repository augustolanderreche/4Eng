1) PENDIENTES CRITICOS (HACER PRIMERO)
==================================================

1.1 Fix driver QMYSQL en Windows ← NUEVO, BLOQUEANTE
Estado actual:
- QMYSQL driver no se carga en la máquina de desarrollo Windows.
- App inicia bien pero login contra VPS no funciona.
- SQLite local funciona correctamente.

Que falta hacer:
- Copiar qsqlmysql.dll al directorio de sqldrivers del build.
- Copiar libmysql.dll al directorio del ejecutable.
- Libmysql.dll viene del MySQL Connector/C o instalación de MariaDB.

Ruta típica en Qt:
- C:\Qt\6.11.0\mingw_64\plugins\sqldrivers\qsqlmysql.dll

--------------------------------------------------
1.2 Seguridad de contraseñas (PENDIENTE en Qt directo)
Estado actual:
- VPS tiene columna password_hash lista (NULL por ahora).
- Qt sigue usando password en texto plano para login.
- FastAPI (cuando se implemente) usará password_hash con bcrypt.

Que falta hacer:
- [FASTAPI] Implementar hash con bcrypt/Argon2id en FastAPI.
- [FASTAPI] Dual-check: intentar password_hash primero, si NULL usar password.
- Qt puede seguir usando password hasta que FastAPI tome el control del login.

--------------------------------------------------
1.3 phpMyAdmin - política definida
Estado actual: Desactivado (404). Correcto para producción.
Que falta: Documentar proceso oficial de activación temporal para mantenimiento.

--------------------------------------------------
1.4 Login/registro directo Qt -> MySQL
Estado actual: Qt consulta DB directo (riesgo de credenciales en cliente).
Plan a futuro: Qt hará requests a Nginx y Nginx enruta a FastAPI (sin acceso directo a DB).
Estado: Aceptable hasta Fase 2, no bloquea consigna actual.

Que falta implementar:

Estado actual: login_attempts se guarda, pero no se muestra en UI.

Que falta:

--------------------------------------------------
2.3 Configurar FastAPI base de autenticación
Que falta hacer:
- Endpoints mínimos (publicados detrás de Nginx):
  - POST /auth/register-user
  - POST /auth/register-company
  - POST /auth/login (actualiza last_login en VPS)
- Validaciones: username/email únicos, role permitido.
- FastAPI usa password_hash (bcrypt).

--------------------------------------------------
2.4 Configurar Nginx delante de FastAPI
- Reverse proxy a FastAPI.
- TLS (Let's Encrypt) en dominio.
- Redirección HTTP -> HTTPS.
- Enrutamiento por path/endpoint hacia puertos internos.
- Exponer solo 80/443 públicamente.

--------------------------------------------------
2.5 Qt consume FastAPI (en lugar de MySQL directo)
- Reemplazar QSqlQuery de login/registro por QNetworkRequest a la API.
- Manejo de errores de red, timeout, reintentos.
- Base URL de Qt apunta a Nginx, no a puertos internos de FastAPI.
