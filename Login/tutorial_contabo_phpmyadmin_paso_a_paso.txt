TUTORIAL COMPLETO: BASE DE DATOS EN VPS CONTABO CON PHPMYADMIN (LISTA PARA FASTAPI/NGINX)
Fecha: 2026-05-11

Objetivo
- Instalar y asegurar MariaDB/MySQL en tu VPS Contabo.
- Instalar phpMyAdmin para gestionar la base por web.
- Crear base de datos y tablas del proyecto (listas para backend FastAPI).
- Dejar phpMyAdmin disponible para que puedas administrar la DB cuando quieras.
- Entregar a tu equipo un SQL listo para importar.

Supuestos
- VPS con Ubuntu 22.04 o 24.04 (si usas Debian es casi igual).
- Tienes acceso SSH al servidor.
- Tienes usuario con permisos sudo.

--------------------------------------------------
PASO 1) ENTRAR AL VPS Y ACTUALIZAR
--------------------------------------------------
Desde tu PC:
ssh TU_USUARIO@TU_IP_VPS

En el VPS:
sudo apt update
sudo apt upgrade -y

--------------------------------------------------
PASO 2) INSTALAR APACHE, MARIADB, PHP Y PHPMYADMIN
--------------------------------------------------
Instala todo junto:
sudo apt install -y apache2 mariadb-server php libapache2-mod-php php-mysql php-mbstring php-zip php-gd php-json php-curl phpmyadmin

Si durante la instalacion de phpMyAdmin te pregunta:
- Web server: selecciona apache2.
- Configurar base con dbconfig-common: selecciona Yes.
- Define password para usuario interno de phpMyAdmin.

Activa extension y reinicia Apache:
sudo phpenmod mbstring
sudo systemctl restart apache2

Comprueba servicios:
sudo systemctl status apache2 --no-pager
sudo systemctl status mariadb --no-pager

--------------------------------------------------
PASO 3) ASEGURAR MARIADB
--------------------------------------------------
Ejecuta:
sudo mysql_secure_installation

Recomendado:
- Cambiar password de root: Yes
- Remove anonymous users: Yes
- Disallow root login remotely: Yes
- Remove test database: Yes
- Reload privilege tables: Yes

--------------------------------------------------
PASO 4) CREAR BASE Y TABLA QUE USA TU APP
--------------------------------------------------
Entra a MariaDB:
sudo mysql -u root -p

Ejecuta este SQL (base lista para lo actual + crecimiento futuro):
CREATE DATABASE IF NOT EXISTS login_db
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
USE login_db;

-- Tabla principal de autenticacion (compatible con tu app actual).
CREATE TABLE IF NOT EXISTS users (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  role ENUM('Usuario','Empresa','Admin') NOT NULL,
  username VARCHAR(50) NOT NULL,
  password VARCHAR(255) NOT NULL,
  email VARCHAR(120) NOT NULL,
  display_name VARCHAR(120) NOT NULL,
  programming_language VARCHAR(50) NULL,
  age INT NULL,
  password_hash VARCHAR(255) NULL,
  is_active TINYINT(1) NOT NULL DEFAULT 1,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT uq_users_username UNIQUE (username),
  CONSTRAINT uq_users_email UNIQUE (email)
);

-- Perfil extendido para ingenieros (usuario rol Usuario).
CREATE TABLE IF NOT EXISTS engineer_profiles (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id BIGINT UNSIGNED NOT NULL,
  first_name VARCHAR(80) NOT NULL,
  last_name VARCHAR(80) NOT NULL,
  phone VARCHAR(30) NULL,
  city VARCHAR(80) NULL,
  country VARCHAR(80) NULL,
  years_experience DECIMAL(4,1) NULL,
  seniority ENUM('Trainee','Junior','SemiSenior','Senior','Lead') NULL,
  linkedin_url VARCHAR(255) NULL,
  github_url VARCHAR(255) NULL,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT uq_engineer_profiles_user UNIQUE (user_id),
  CONSTRAINT fk_engineer_profiles_user FOREIGN KEY (user_id)
    REFERENCES users(id) ON DELETE CASCADE ON UPDATE CASCADE
);

-- Perfil extendido para empresas (usuario rol Empresa).
CREATE TABLE IF NOT EXISTS company_profiles (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id BIGINT UNSIGNED NOT NULL,
  company_name VARCHAR(140) NOT NULL,
  tax_id VARCHAR(40) NULL,
  website VARCHAR(255) NULL,
  industry VARCHAR(100) NULL,
  company_size ENUM('1-10','11-50','51-200','201-500','500+') NULL,
  contact_name VARCHAR(120) NULL,
  contact_phone VARCHAR(30) NULL,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT uq_company_profiles_user UNIQUE (user_id),
  CONSTRAINT fk_company_profiles_user FOREIGN KEY (user_id)
    REFERENCES users(id) ON DELETE CASCADE ON UPDATE CASCADE
);

-- CVs cargados por ingenieros.
CREATE TABLE IF NOT EXISTS cv_documents (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  engineer_user_id BIGINT UNSIGNED NOT NULL,
  file_name VARCHAR(255) NOT NULL,
  file_url VARCHAR(500) NOT NULL,
  file_sha256 CHAR(64) NULL,
  extracted_text MEDIUMTEXT NULL,
  parsed_skills JSON NULL,
  uploaded_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_cv_documents_engineer FOREIGN KEY (engineer_user_id)
    REFERENCES users(id) ON DELETE CASCADE ON UPDATE CASCADE,
  INDEX idx_cv_documents_engineer_user_id (engineer_user_id)
);

-- Solicitudes de trabajo creadas por empresas.
CREATE TABLE IF NOT EXISTS job_posts (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  company_user_id BIGINT UNSIGNED NOT NULL,
  title VARCHAR(150) NOT NULL,
  description MEDIUMTEXT NOT NULL,
  required_skills JSON NULL,
  min_years_experience DECIMAL(4,1) NULL,
  location_mode ENUM('Remoto','Hibrido','Presencial') NOT NULL DEFAULT 'Remoto',
  city VARCHAR(80) NULL,
  country VARCHAR(80) NULL,
  status ENUM('Abierta','Pausada','Cerrada') NOT NULL DEFAULT 'Abierta',
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT fk_job_posts_company FOREIGN KEY (company_user_id)
    REFERENCES users(id) ON DELETE CASCADE ON UPDATE CASCADE,
  INDEX idx_job_posts_company_user_id (company_user_id),
  INDEX idx_job_posts_status (status)
);

-- Postulaciones de ingenieros a ofertas.
CREATE TABLE IF NOT EXISTS applications (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  job_post_id BIGINT UNSIGNED NOT NULL,
  engineer_user_id BIGINT UNSIGNED NOT NULL,
  cv_document_id BIGINT UNSIGNED NULL,
  status ENUM('Postulada','EnRevision','Seleccionada','Rechazada') NOT NULL DEFAULT 'Postulada',
  applied_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT fk_applications_job_post FOREIGN KEY (job_post_id)
    REFERENCES job_posts(id) ON DELETE CASCADE ON UPDATE CASCADE,
  CONSTRAINT fk_applications_engineer FOREIGN KEY (engineer_user_id)
    REFERENCES users(id) ON DELETE CASCADE ON UPDATE CASCADE,
  CONSTRAINT fk_applications_cv FOREIGN KEY (cv_document_id)
    REFERENCES cv_documents(id) ON DELETE SET NULL ON UPDATE CASCADE,
  CONSTRAINT uq_applications_job_engineer UNIQUE (job_post_id, engineer_user_id),
  INDEX idx_applications_engineer_user_id (engineer_user_id),
  INDEX idx_applications_status (status)
);

-- Resultado de analisis IA sobre cada postulacion.
CREATE TABLE IF NOT EXISTS ai_evaluations (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  application_id BIGINT UNSIGNED NOT NULL,
  model_name VARCHAR(80) NOT NULL,
  score DECIMAL(5,2) NULL,
  recommendation ENUM('NoRecomendada','RecomendadaConCapacitacion','Recomendada') NOT NULL,
  missing_skills JSON NULL,
  summary TEXT NULL,
  raw_response LONGTEXT NULL,
  evaluated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_ai_evaluations_application FOREIGN KEY (application_id)
    REFERENCES applications(id) ON DELETE CASCADE ON UPDATE CASCADE,
  INDEX idx_ai_evaluations_application_id (application_id),
  INDEX idx_ai_evaluations_recommendation (recommendation)
);

-- Capacitaciones sugeridas por IA para cubrir brechas.
CREATE TABLE IF NOT EXISTS training_recommendations (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  ai_evaluation_id BIGINT UNSIGNED NOT NULL,
  skill_name VARCHAR(120) NOT NULL,
  recommendation_text TEXT NOT NULL,
  provider VARCHAR(120) NULL,
  resource_url VARCHAR(500) NULL,
  estimated_hours INT NULL,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_training_recommendations_ai FOREIGN KEY (ai_evaluation_id)
    REFERENCES ai_evaluations(id) ON DELETE CASCADE ON UPDATE CASCADE,
  INDEX idx_training_recommendations_ai_evaluation_id (ai_evaluation_id)
);

-- Acciones administrativas (auditoria basica).
CREATE TABLE IF NOT EXISTS admin_audit_logs (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  admin_user_id BIGINT UNSIGNED NOT NULL,
  action VARCHAR(120) NOT NULL,
  target_type VARCHAR(60) NULL,
  target_id BIGINT UNSIGNED NULL,
  details JSON NULL,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_admin_audit_logs_admin FOREIGN KEY (admin_user_id)
    REFERENCES users(id) ON DELETE CASCADE ON UPDATE CASCADE,
  INDEX idx_admin_audit_logs_admin_user_id (admin_user_id),
  INDEX idx_admin_audit_logs_created_at (created_at)
);

Validaciones/peticiones recomendadas para backend futuro:
1) Registro:
   - username y email unicos.
   - role solo Usuario, Empresa o Admin.
   - Empresa requiere company_profiles.
   - Usuario requiere engineer_profiles.

2) CV:
   - solo Usuario puede subir a cv_documents.
   - validar tamano, extension y hash duplicado.

3) Vacantes:
   - solo Empresa puede crear/editar job_posts.
   - no permitir editar vacante Cerrada salvo Admin.

4) Postulaciones:
   - no duplicar (job_post_id, engineer_user_id).
   - solo vacantes Abiertas aceptan postulacion.

5) IA:
   - cada ai_evaluations debe apuntar a applications real.
   - recommendation y score se guardan para trazabilidad.

6) Admin:
   - toda accion sensible queda en admin_audit_logs.

--------------------------------------------------
PASO 5) CREAR USUARIOS MYSQL (TU ROL + EQUIPO BACKEND)
--------------------------------------------------
Aun dentro de MariaDB, crea usuarios segun uso real:

1) Usuario para TI (administrar DB desde phpMyAdmin):
CREATE USER 'dbadmin'@'localhost' IDENTIFIED BY 'cambia_esto_admin';
GRANT ALL PRIVILEGES ON login_db.* TO 'dbadmin'@'localhost';

2) Usuario para el backend (FastAPI en el VPS):
CREATE USER 'backend_app'@'localhost' IDENTIFIED BY 'cambia_esto_backend';
GRANT SELECT, INSERT, UPDATE, DELETE ON login_db.* TO 'backend_app'@'localhost';

FLUSH PRIVILEGES;

Notas importantes:
- Para tu caso actual NO necesitas usar tu IP publica de PC en MySQL.
- Si FastAPI y MariaDB corren en el mismo VPS, localhost es lo correcto.
- Solo usa 'usuario'@'X.X.X.X' o 'usuario'@'%' si realmente habra conexion remota.

Sal de MariaDB:
EXIT;

--------------------------------------------------
PASO 6) HABILITAR PHPMYADMIN EN APACHE
--------------------------------------------------
Prueba en navegador:
http://TU_IP_VPS/phpmyadmin

Si no abre, enlaza config manualmente:
sudo ln -s /etc/phpmyadmin/apache.conf /etc/apache2/conf-available/phpmyadmin.conf
sudo a2enconf phpmyadmin
sudo systemctl reload apache2

--------------------------------------------------
PASO 7) ASEGURAR PHPMYADMIN (MUY IMPORTANTE)
--------------------------------------------------
1. Firewall basico:
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

2. Ideal: restringir acceso a phpMyAdmin por IP.
Puedes agregar reglas en Apache para permitir solo tu IP.

Si tu IP cambia seguido y quieres entrar siempre, usa esta alternativa:
- Mantener acceso por HTTPS + password fuerte + autenticacion extra de Apache.

Autenticacion extra (recomendada):
sudo apt install -y apache2-utils
sudo htpasswd -c /etc/phpmyadmin/.htpasswd tuusuarioadmin

Edita /etc/phpmyadmin/apache.conf y dentro de <Directory /usr/share/phpmyadmin> agrega:
AuthType Basic
AuthName "Restricted phpMyAdmin"
AuthUserFile /etc/phpmyadmin/.htpasswd
Require valid-user

Aplica cambios:
sudo systemctl reload apache2

3. HTTPS con Certbot (si tienes dominio):
sudo apt install -y certbot python3-certbot-apache
sudo certbot --apache -d tu-dominio.com

4. No expongas 3306 globalmente si no es necesario.
Si puedes, usa tunel SSH para administrar DB y mantener 3306 cerrado.

--------------------------------------------------
PASO 8) (OPCIONAL) ABRIR MYSQL REMOTO
--------------------------------------------------
Solo si algun servicio FUERA del VPS se conectara directo a MySQL.
Si FastAPI correra en el mismo VPS, NO lo necesitas.

Edita:
sudo nano /etc/mysql/mariadb.conf.d/50-server.cnf

Busca y cambia:
bind-address = 127.0.0.1
por
bind-address = 0.0.0.0

Reinicia MariaDB:
sudo systemctl restart mariadb

Firewall: permite 3306 solo desde tu IP de desarrollo (recomendado):
sudo ufw allow from X.X.X.X to any port 3306 proto tcp

Evita esta regla en produccion:
sudo ufw allow 3306/tcp

Import en servidor destino (si lo necesitan):

--------------------------------------------------
SOLUCION DE PROBLEMAS
--------------------------------------------------
A) Error en app: Driver not loaded / QMYSQL
- Falta plugin QMYSQL compatible con tu version de Qt.
- Instala/compila el plugin correspondiente a tu kit.

Si no usaras Qt directo contra MySQL, puedes ignorar este punto.

B) Error acceso denegado para backend_app o dbadmin
- Revisa host permitido del usuario MySQL (localhost si corre en el VPS).
- Revisa password y grants.

C) phpMyAdmin no abre
- Verifica Apache activo.
- Verifica configuracion enlazada de phpMyAdmin.
- Revisa firewall UFW (80/443).

D) Timeout al conectar desde backend externo (si aplica)
- Revisa si 3306 esta permitido desde el origen correcto.
- Revisa bind-address.
- Revisa reglas de firewall externo de Contabo si aplican.

E) No puedes entrar siempre a phpMyAdmin
- Usa dominio + HTTPS.
- Si tu IP cambia, evita restriccion estricta por IP y usa autenticacion extra
  con .htpasswd + password fuerte.
- Mantiene 3306 cerrado si no hay necesidad de acceso remoto.

--------------------------------------------------
RECOMENDACION FINAL DE SEGURIDAD
--------------------------------------------------
Tu app actual guarda password en texto plano. Para produccion:
- Usa hash de password (Argon2, bcrypt o PBKDF2).
- Idealmente no conectes Qt directo a MySQL publica.
- Usa API intermedia (backend) y DB privada.
