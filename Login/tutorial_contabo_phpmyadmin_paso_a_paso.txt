7) CONFIGURAR ACCESO PARA FASTAPI Y QT
==================================================
Usuarios recomendados:

A) Para Qt (lectura y validación de login):
```bash
mysql -u root -p login_db -e "
CREATE USER IF NOT EXISTS 'qtuser'@'127.0.0.1' IDENTIFIED BY 'qtpass';
GRANT SELECT, INSERT ON login_db.users TO 'qtuser'@'127.0.0.1';
FLUSH PRIVILEGES;
"
```

B) Para FastAPI (completo):
```bash
mysql -u root -p login_db -e "
CREATE USER IF NOT EXISTS 'backend_app'@'127.0.0.1' IDENTIFIED BY 'backend_pass';
GRANT SELECT, INSERT, UPDATE, DELETE ON login_db.* TO 'backend_app'@'127.0.0.1';
FLUSH PRIVILEGES;
"
```

C) Para Admin (todas las operaciones):
```bash
mysql -u root -p login_db -e "
CREATE USER IF NOT EXISTS 'dbadmin'@'127.0.0.1' IDENTIFIED BY 'admin_secure_pass';
GRANT ALL PRIVILEGES ON login_db.* TO 'dbadmin'@'127.0.0.1';
FLUSH PRIVILEGES;
"
```

Nota:
- Si se necesita acceso remoto, reemplazar 127.0.0.1 por % (pero es menos seguro).
- Si se necesita acceso SSH tunnel, seguir usando 127.0.0.1 (más seguro).

El Qt actual lee estos valores (ver databaseconfig.h):

Variables esperadas:
- LOGIN_DB_HOST (default: 127.0.0.1)
- LOGIN_DB_PORT (default: 3306)
- LOGIN_DB_NAME (default: login_db)

11) TROUBLESHOOTING
==================================================
A) "Access denied" al conectarse:
   - Verifica USER y PASSWORD en mysql command.
   - Revisa que el usuario tenga permisos con SHOW GRANTS FOR 'usuario'@'host';

B) "Table 'login_db.users' doesn't exist":
   - Si la tabla se llama "usuarios", renómbrala primero:
     RENAME TABLE usuarios TO users;
   - Luego ejecuta el script nuevamente.

C) "Duplicate entry for key":
   - Significa que el script intentó crear algo que ya existe.
   - Es normal con CREATE TABLE IF NOT EXISTS.
   - Los datos existentes NO se pierden.

D) "Constraint Fails":
   - Posible: datos en users que no encajan en la nueva estructura.
   - Solución: revisar y corregir datos antes de guardar constraints.

E) Qt no puede conectarse:
   - Verifica: mysql -h HOST -u USER -p PASS -e "SELECT 1"
   - Verifica: variables de entorno están configuradas.
   - Verifica: puerto 3306 está abierto entre cliente y VPS (SSH tunnel recomendado).
