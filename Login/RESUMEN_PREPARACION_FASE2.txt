PASO 3: MIGRACIÓN GRADUAL password → password_hash
────────────────────────────────────────────────
Quién: FastAPI dev + Qt dev
Cuándo: Cuando FastAPI esté listo

Estrategia:
- Qt actual: sigue usando password (no toca password_hash)
- FastAPI nuevo: usa password_hash para nuevas cuentas
- Usuarios viejos: FastAPI verifica password si password_hash es NULL
- Primera vez que user viejo loguea en FastAPI:
  → Se migra a bcrypt automáticamente
  → FastAPI actualiza password_hash
  → Siguiente login usa password_hash

Esto permite transición sin romper nada.

P: ¿Es seguro ejecutar estos cambios ahora?
R: Sí. Solo agregan columna NULL e índices. No elimina nada, no modifica datos.

P: ¿Qt se va a romper?
R: No. Qt sigue igual. Los índices aceleran queries, punto.

R: Hoy, mañana, cuando quieras. No afecta a Qt.

R: Sí. DROP INDEX idx_name; DROP COLUMN last_login; vuelve atrás.

P: ¿Afecta a phpMyAdmin?
R: No. phpMyAdmin solo navega la BD, no se rompe.
