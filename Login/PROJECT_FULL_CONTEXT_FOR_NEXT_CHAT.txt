1) QUICK SUMMARY
==================================================
This is a desktop login/registration app built with Qt Widgets.
Fase 1 actual conecta directo a MySQL/MariaDB (VPS) usando QMYSQL.
Fase 2 objetivo: Qt deja conexión directa y consume FastAPI vía Nginx (reverse proxy).
También usa SQLite local para session management, offline cache y bruteforce protection.

Users can:
- Log in with username/password.
- Register as Empresa (company) or Usuario (individual).
- Be redirected after login by role:
  - role == "Empresa" -> EmpresaWindow
  - any other role -> UserWindow

En Fase 1 las credenciales están en variables de entorno vía databaseconfig.h.
En Fase 2 las credenciales de DB quedan solo del lado backend (FastAPI), no en Qt.

LOCAL SQLITE (NEW - 2026-05-15):
- Managed via localdbconfig.h
- Path: %APPDATA%/4eng-Login/local.db (created automatically on first run)
- Handles: sessions, login history, bruteforce blocking, offline cache,
           user preferences, audit log, sync metadata

IMPORTANT OPERATIONS STATUS (VPS):
- phpMyAdmin was enabled and tested successfully from browser.
- Later it was intentionally disabled in Apache for security (current result: /phpmyadmin returns 404 Not Found).
- UFW now allows 80/443 (web access control moved to Apache config state).
- Docker exists on VPS with project containers observed (vps-api-1, vps-db-1); phpMyAdmin in this setup was not served from Docker.

4) UI FLOW / NAVIGATION
==================================================
Entry point:
- main.cpp creates QApplication, sets Fusion style and a global dark stylesheet,
  then shows MainWindow.

MainWindow behavior (UPDATED):
- On startup: checks SQLite for active session → auto-login if found.
- If no session: displays login form (username, password).
- Buttons: Entrar, Registrar.
- Entrar -> attemptLogin().
- Registrar -> opens RegisterWindow.

Login logic in MainWindow (UPDATED 2026-05-15):
1. Validate non-empty fields.
2. checkAndHandleFailedAttempts(username) → check local SQLite block.
   - If blocked (>=5 failed attempts): show "Bloqueado X segundos", abort.
3. validateCredentials(username, password) → check VPS MySQL.
4. If VPS fails: incrementFailedAttempts() in SQLite, log attempt.
5. If VPS success:
   - role = getRole(username), displayName = getDisplayName(username)
   - saveLoginToLocal() → save session in SQLite, reset failed attempts
   - role == "Empresa" => openEmpresaWindow(displayName)
   - else => openUserWindow(displayName)

Auto-login (NEW):
- setupAutoLogin() called on startup.
- Checks SQLite sessions WHERE is_active=1.
- If found: skips login form, goes directly to role window.

Background sync (NEW):
- QTimer every 5 minutes: performBackgroundSync() → syncOfflineCache().
- QTimer inactivity 30 min: closes session automatically.

Close event (NEW):
- closeEvent() triggers syncOfflineCache() + logAction("app_closed") + closeSession().

RegisterWindow behavior:
- QTabWidget with two tabs: Empresa / Usuario
- Empresa: username, password, email
- Usuario: firstName, lastName, username, password, language, email, age (10-120)
- Validation: no empty fields, email contains '@', username unique
- Inserts to VPS MySQL users table

Post-login windows:
- EmpresaWindow: welcome label "Hola Empresa <displayName>"
- UserWindow: welcome label "Hola Usuario <displayName>"

7) FIX PENDIENTE: QMYSQL DRIVER EN WINDOWS
==================================================
Problema: "can not load requested driver 'QMYSQL'"
Causa: qsqlmysql.dll no está en la ruta de plugins de Qt.

Solución:
1. Buscar qsqlmysql.dll en la instalación de Qt:
   C:\Qt\6.11.0\mingw_64\plugins\sqldrivers\qsqlmysql.dll
2. Copiar al directorio del ejecutable:
   build/Desktop_Qt_6_11_0_MinGW_64_bit-Debug/debug/sqldrivers/
3. También copiar libmysql.dll al directorio del ejecutable.
4. Libmysql.dll se obtiene de MySQL Connector/C o de la instalación de MariaDB.

Sin este fix: Login contra VPS no funciona, SQLite local sí funciona.

8) SECURITY STATUS (IMPORTANT)
==================================================
Current state:
- Passwords are stored and checked as plain text.
- Direct DB connection from client app to VPS DB (current Qt phase).
- Qt currently does NOT use SHA2, bcrypt, Argon2, or password_hash for auth.
- phpMyAdmin endpoint currently disabled (404) to prevent public access.

Security implications:
- Not production-safe for real users.
- Vulnerable if DB credentials are leaked from client environment.
- Test credentials in plain text must be replaced/removed before production.

Recommended improvements (future):
1. Hash passwords (Argon2/bcrypt/PBKDF2) + salt.
2. Move auth to backend API (server-side), avoid direct client DB access.
3. Restrict DB user permissions and host allowlist tightly.
4. Use TLS or secure tunnel where possible.
5. Keep phpMyAdmin disabled by default; enable only for maintenance windows.
